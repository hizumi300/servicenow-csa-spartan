# PersonalClaw Instructions

PersonalClaw is a small control repository for using Codex as a personal, local, full-access agent.

## Repository Role

- Treat this repository as a standing framing layer, not as a platform that should expand by default.
- The OpenClaw comparison is conceptual. Do not treat it as a request to recreate OpenClaw, roleplay a separate system, or grow this repository into a framework.
- This repository often provides context for work that belongs somewhere else.
- If the user's task belongs in another directory or repository, do the work there and leave PersonalClaw unchanged unless a change here is directly useful.

## Default Behavior

- Treat this repository as intentionally minimal and potentially already complete.
- Prefer the narrowest useful interpretation of the user's request.
- Do not add scaffolding, frameworks, placeholder code, speculative architecture, or organizational structure unless explicitly requested.
- Prefer no change over speculative change, and prefer a small durable change over a larger structural one.
- If you are unsure whether a new file belongs here, default to not creating it.

## What Belongs Here

- `README.md` explains the idea to humans.
- `AGENTS.md` carries the standing repository instructions for Codex.
- Additional files should exist only when they improve repeated real use of PersonalClaw itself.

## Editing Style

- Keep changes direct, compact, and concrete.
- Preserve the idea that a low file count can be the correct finished state.
